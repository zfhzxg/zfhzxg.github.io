Hadoop 2.X管理与开发

发表于 2018-09-05

###hadoop的起源与背景知识

  • 一、什么是大数据

    举例:

    1.电商的推荐系统(可能会用到推荐算法:协同过滤,ALS,逻辑回归…)
    2.天气预报

    核心的问题:数据的存储,数据的计算(不是算法)

    数据的存储:采用分布式的文件系统HDFS(hadoop Distributed file system)

    数据的计算:采用分布式的计算MapReduce,Spark(RDD:弹性分布式数据集)

  • 二、数据仓库和大数据

    传统方式:搭建数据仓库(Data Werehouse)来解决大数据的问题

    数据仓库就是一个数据库(Orcale,MySQL,MS)
    数据仓库和大数据一般只做查询(分析)
    搭建数据仓库的过程

    数据源RDBMS:关系型数据库(结构化数据)/文本数据/其他数据
    利用ETL抽取/转化/加载数据后搭建数据仓库(保存为原始数据)
    分析处理数据(SQL,PL/SQL,JDBC)
    经过分析后搭建数据集市(hr,sales)
    提供给hr系统或销售系统

    Hadoop和Spark都可以看成是数据仓库的一种实现

    从ETL到搭建数据集都可以用Hadoop和Spark所提供的方式解决,也可以用传统方式解决
    hadoop中使用sqoop和Flume组件完成ETL
    hadoop中使用HDFS存储数据(或存入基于HDFS之上的HBase或Hive数据仓库)
    hadoop中使用MapReduce(java程序)或Spark(Scala程序,只有数据计算,没有数据存储)或SQL处理数据
    *大数据的终极目标是使用SQL处理数据
    处理数据后使用HDFS或NoSQL:Redis存储到数据集市

  • 三、OLTP和OLAP

    OLTP:Online Transaction processing 联机事务处理，指：insert、update、delete —> 事务

    OLAP:Online Analytic Processing 联机分析处理，一般制作查询 —> 数据仓库就是一种OLAP的应用系统

    Hadoop、Spark看成是一种数据仓库的解决方案

    数据仓库（查询）不支持事务

  • 四、Google的基本思想:三篇论文

    (一)GFS（Google File System） —- HDFS（Hadoop Distributed File System）：分布式文件系统

    1.分布式文件系统
    2.大数据的存储问题
    3.HDFS中，记录数据的位置信息（元信息） —- 采用倒排索引（Reverted Index）

    (1)什么是索引？Index
    *CREATE INDEX创建索引
    *索引就是一个记录（Oracle中索引表保存的是有规律的行地址）
    *通过索引可以找到对应的数据
    (2)什么是倒排索引？
    *最简单的倒排索引：单词表（wordID，word，index）
    (3)NameNode（主节点，名称节点）是整个HDFS的管理员，和SecondaryNameNode（第二名称节点）同处一台主机，负责管理DateNode（从节点，数据节点），并不负责存储，与DateNode（从节点，数据节点）构成HDFS
    *

    (二)MapReduce：分布计算模型，问题来源PageRank

    (三)BigTable：大表 —- NoSQL数据库：HBase
